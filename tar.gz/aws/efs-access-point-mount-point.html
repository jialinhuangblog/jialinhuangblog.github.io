mount point (EC2’s)

A mount point is a local path on your instance or computing resource that connects to a remote disk.

access point (EFS’s)

Access points are used by EFS for categorizing and restricting external access.

For example, EFS can have multiple access points, and other instances' mount points can point to specific access points.

Access points are for differentiating access restrictions. Actually, the access point step isn't always necessary.

Example

Common Steps

  1. Create EFS
  1. Create mount targets in different VPCs (must)
  1. Define access points (optional)
  1. Configure EC2 mount points to point to access points

Two Ways

  1. Simple: EC2 → Mount Point → Mount Target → EFS
  1. Classified: EC2 → Mount Point → Access Point → Mount Target → EFS

One EFS can be divided into multiple Access Points:

EFS File System
├── /app1-files     <- Access Point 1 (restricted to access only this directory)
├── /app2-files     <- Access Point 2 (restricted to access only this directory)
└── /shared-files   <- Access Point 3 (shared access)

Multiple instances use different mount points, which you name yourself.

/mnt is a conventional name for "temporary mounts" in the Linux Filesystem Hierarchy Standard (FHS)

EC2 instances

APP1 Team (use Access Point 1)
EC2-A: /mnt/app1
EC2-B: /var/app1-data
EC2-C: /application/data

APP2 Team (use Access Point 2)
EC2-D: /mnt/app2
EC2-E: /data/app2
EC2-F: /var/www/app2-files

Shared Resource Team (use Access Point 3)
EC2-G: /shared/data
EC2-H: /mnt/shared
EC2-I: /var/shared-resources
EC2-J: /opt/shared

Mount point and access point connection example:

#!/bin/bash
# ec2
yum install -y amazon-efs-utils
mkdir -p /mnt/efs
mount -t efs -o tls,accesspoint=access-point-id file-system-id:/ /mnt/efs
