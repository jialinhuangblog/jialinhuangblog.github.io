TL;DR

COPY is the safer and preferred option.

ADD

Suitable for scenarios where you need to fetch files from remote sources and automatically extract them

Not recommended.

# At the start of docker, ./ is already limited

# aFolder will be scattered directly in your pwd
ADD http://otherplace.com/aFolder .

# aFolder itself will create a new directory => /aFolder
ADD http://otherplace.com/aFolder /aFolder

COPY (üëç)

Ideal for simple local file copying

More predictable, recommended.

# RELATIVE: <WORKDIR>/relativeDir/ 
COPY test.txt relativeDir/

# ABSOLUTE: /absoluteDir/
COPY test.txt /absoluteDir/

https://stackoverflow.com/questions/24958140/what-is-the-difference-between-the-copy-and-add-commands-in-a-dockerfile

Similarities

Both COPY and ADD cannot perform operations that go beyond their scope, such as using ../

Example.

FROM alpine:latest

COPY ../file.txt /app/

CMD ["echo", "This Dockerfile will fail to build"]

This Dockerfile will fail to build because it attempts to copy a file from outside the build context.
