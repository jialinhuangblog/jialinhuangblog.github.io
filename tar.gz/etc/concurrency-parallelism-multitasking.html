Multitasking

The concept of running multiple tasks, which can be implemented on both single and multiple processors.

  • Time Slicing: Like a super scheduler dividing a day into 5-minute slots, but here it's milliseconds.
  • Time Sharing: Using fast context switching to make each task think it's the only one running.
  • Time Sharing is the concept, Time slicing is one specific strategy of Time sharing.

Multitasking includes:

  1. multiprocessing (parallelism on multiple CPUs)
  1. multithreading (can be concurrent on a single CPU or parallel on multiple CPUs)

The meaning depends on context. In systems, multitasking usually refers to multiprocessing. In programming, it often means multithreading.

Multiprocessing

The ability to run multiple tasks simultaneously using multiple processors (CPUs).

Always concurrent, but parallel only when multiple CPUs are actually used.

Synchronous/sync

The definition varies in different situations:

  • Happening, existing, or arising at precisely the same time?
  • Operations tasks are performed one at a time?
  • As sequential

Asynchronous/async

A programming paradigm where operations can be initiated without waiting for previous operations to complete, allowing for non-blocking execution.

Events occur independently of the main program flow, allowing the program to continue running without waiting for an operation to complete.

Async programming is often used for I/O-bound tasks to improve application responsiveness and resource utilization.

Implementations include:

  1. Callbacks
  1. Promises
  1. Coroutines

Asynchronous vs. Threading

Same: to achieve concurrency

Many languages implement async features without using multiple threads:

  • JavaScript, for example, is single-threaded and uses an event loop for async operations.
  • Python's asyncio framework builds async functionality on top of a single thread.

Language-Specific Implementations:

  • JavaScript/Node.js:
    • General: Primarily single-threaded, using an event loop for asynchronous operations through callbacks, Promises, and async/await.
    • Worker Threads (introduced in Node.js 10.5.0):
      • limited multithreading capabilities within a single process in javascript runtime. Allow running JavaScript in background threads for CPU-intensive tasks.
    • Node.js Child Processes: Enable true multiprocessing by spawning separate OS processes, allowing full utilization of multi-core systems.
  • Python: The asyncio library provides async capabilities using coroutines, all running on a single thread.

Node.js, features like Worker Threads and Child Processes can be considered "out-of-band" mechanisms. They provide additional concurrency and parallelism options beyond the core single-threaded model, but their usage is optional and specific to certain use cases.

Parallelism

Must have CPUs to do it.

Concurrency

Concurrency is a blueprint: how to organize and manage multiple tasks that might happen at the same time, whether or not they actually run simultaneously.

It's an abstract concept that can be achieved through multiple ways:

  • Multitasking
  • Multiprocessing
  • Multithreading
  • Async
  • Sync
  • One CPU? OK
  • Many CPUs? Great

Although some programming languages like Python provide multiprocessing APIs that can be used in user-level code, their core implementation still relies on system-level operations.
