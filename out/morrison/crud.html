這是 HTTP 溝通的方式，但並沒有侷限說不能用 POST 取得 resource，而是大家約定好的共識。

  • GET 取得資料，aka read
  • POST 通常是新增資料， aka create
  • UPDATE/PATCH 通常會用來新增資料， patch 是部分改動， update 是整個 resource 換掉，aka update
  • DELETE 通常會用來移除 specific resouce，aka delete
  • OPTION 瀏覽器的行為會發出 preflight，preflight 是 CORS 規範之一，為的是怕跨資源的會產生預期外的影響，所以會發一個 OPTIONS 問問看後端
